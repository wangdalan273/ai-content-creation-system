# 微信公众号文章结构模板库

---

## 模板1：问题解决型（B 技术博客）

### 适用场景
解决用户痛点、提供解决方案的文章

### 结构框架
```
【标题】如何解决XXX问题？这X个方法很有效

【开头】（50-100字）
- 从具体场景/踩坑经历切入
- 不铺垫，直接进入

【主体】（1200-2000字）
小标题1：问题分析
- 问题产生的原因
- 问题的影响
- 具体细节（数字、截图）

小标题2：解决方案一
- 具体方法说明
- 实施步骤 + 代码/命令
- 踩坑注意点

小标题3：解决方案二
- （同上结构）

小标题4：进阶技巧
- 更深入的操作
- 个性化配置建议

【结尾】（100字）
- 个人总结（不是AI总结）
- 行动引导
```

### 示例
```
标题：Cursor 代码补全总给出错误建议？3个设置改完就好了

开头：
说实话，这个功能我第一次用完全搞反了，白折腾了一个下午。
后来发现不是工具的问题，是三个默认设置在捣乱。

【以下按框架展开】
```

## 模板2：观点分析型（E 观点洞察）

### 适用场景
行业洞察、趋势分析、深度评论类文章

### 结构框架
```
【标题】关于XXX，这些变化值得关注

【开头：事件锚点】（100-150字）
- 真实事件/新闻切入，具体到人名/公司/数字
- 不铺垫，直接给事件细节

【主体】（1200-2000字）
小标题1：这件事说明了什么
- 核心矛盾是什么
- 为什么值得注意

小标题2：多维分析
- 角度一：+ 案例
- 角度二：+ 案例
- 角度三：+ 案例

小标题3：我的判断
- 明确表态
- 说清楚为什么这么想
- 允许"我不确定"

【结尾】（50-100字）
- 给读者一个可行动的方向
- 或留下思考空间（不喊口号）
```

### 示例
```
标题：配音演员发现AI在用自己的声音接活，背后的问题比你想的大

开头：
美国有个配音演员，去年收入还不错，今年突然发现有人在用她的声音接活——AI克隆的，她完全不知情。

这不是个案。

【以下按框架展开】
```

## 模板3：对比评测型（D 评测对比）

### 适用场景
产品对比、方案选择、优劣分析

### 结构框架
```
【标题】XXX vs XXX，到底该选哪个？

【开头】（50-100字）
- 为什么做这个对比
- 先给结论/推荐（场景化的）

【主体】（1200-2000字）
小标题1：评测维度说明
- 从哪几个角度比，为什么选这些维度

小标题2：逐维度对比
| 维度 | A方案 | B方案 | 胜出 |
|------|-------|-------|------|
| ... | ... | ... | ... |
文字补充每个维度的使用体验

小标题3：场景推荐
- 你是XX场景 → 推荐A，因为...
- 你是YY场景 → 推荐B，因为...

【结尾】（100字）
- 个人选择 + 原因
- 引导分享你的选择
```

### 示例
```
标题：Cursor 和 Claude Code 各用了一个月，我留了一个

开头：
结论先说：如果你是独立开发者选 Claude Code，如果你在团队里选 Cursor。
原因不是网上说的那些，是我自己的体感。

【以下按框架展开】
```

## 模板4：干货教程型（B 技术博客）

### 适用场景
技能教学、操作指南、实用技巧

### 结构框架
```
【标题】X步教你掌握XXX

【开头】（50-100字）
- 这技能有什么用
- 你的预期收获

【主体】（1500-2500字）
小标题1：准备工作
- 工具准备
- 环境准备（代码块展示命令）

小标题2：核心步骤
- 步骤一：详细操作 + 截图 + 代码
- 步骤二：（同上）
- 步骤三：（同上）
- 每步标注常见错误和解决方法

小标题3：进阶技巧
- 进阶用法
- 效率提升技巧

小标题4：常见问题
- Q1: ...
- A1: ...
- （3-5个高频问题）

【结尾】（100字）
- 学习建议
- 实践方向
```

### 示例
```
标题：4步搭建你的第一个 AI Agent，跟着做就行

开头：
我第一次搭 Agent 的时候，光配环境就折腾了半天。
其实核心就4步，跑通了一遍大概15分钟。

【以下按框架展开】
```

## 使用建议

1. **根据文章类型选模板** — B 用模板1/4，D 用模板3，E 用模板2
2. **模板可混合** — 比如教程文章中间加一段对比评测
3. **灵活调整** — 不要生搬硬套，保持自然
4. **参考 writing-style.md** — 确保语气符合对应风格
