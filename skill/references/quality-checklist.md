# 微信公众号文章质量审核清单

## 一、内容准确性检查

### 1.1 事实核查
- [ ] 所有数据来源明确且可追溯
- [ ] 引用的统计数字准确无误
- [ ] 时间、地点、人物等基础信息正确
- [ ] 专业术语使用准确
- [ ] 没有虚假信息或误导性内容

### 1.2 来源验证
- [ ] 引用内容注明了出处
- [ ] 引用来源权威可靠
- [ ] 数据是最新的或已说明数据时效

### 1.3 逻辑一致性
- [ ] 文章论点前后一致
- [ ] 论证过程合理有据
- [ ] 没有自相矛盾的表述
- [ ] 结论与论据匹配

---

## 二、语言规范性检查

### 2.1 文字基础
- [ ] 无错别字
- [ ] 无病句
- [ ] 标点符号使用正确
- [ ] 中英文之间有空格
- [ ] 专有名词拼写正确

### 2.2 语言风格
- [ ] 读起来像在跟朋友聊天
- [ ] 无 AI 套话（"在当今"、"综上"、"值得注意的是"）
- [ ] 无"不是A而是B"句式滥用
- [ ] 有态度有立场，不骑墙
- [ ] 句式长短适中，节奏流畅

### 2.3 表达清晰度
- [ ] 每句话意思明确
- [ ] 专业术语有必要解释
- [ ] 复杂概念用通俗语言说明
- [ ] 无冗余废话

---

## 三、结构完整性检查

### 3.1 标题质量
- [ ] 标题与内容高度匹配
- [ ] 标题长度适中（13-20字）
- [ ] 包含核心关键词
- [ ] 不是标题党

### 3.2 文章结构
- [ ] 开头引人入胜（50-200字）
- [ ] 主体逻辑清晰，层次分明
- [ ] 使用小标题分段（3-5个）
- [ ] 段落长度适中（手机端3-5行）
- [ ] 结尾有力

### 3.3 内容完整性
- [ ] 开头-主体-结尾结构完整
- [ ] 每个观点都有充分论证
- [ ] 没有遗漏重要信息

---

## 四、可读性检查

### 4.1 视觉呈现
- [ ] 段落之间有适当留白
- [ ] 重点内容有加粗强调
- [ ] 小标题醒目易识别
- [ ] 配图位置合理

### 4.2 阅读体验
- [ ] 文章节奏感好，不枯燥
- [ ] 避免大段文字堆砌
- [ ] 数据和案例穿插得当
- [ ] 易于快速扫读

---

## 五、价值性检查

### 5.1 信息价值
- [ ] 提供了新知识或新观点
- [ ] 有实用的方法或技巧
- [ ] 帮助读者解决实际问题
- [ ] 引发思考或启发

### 5.2 独特性
- [ ] 有独特的观点或角度
- [ ] 不是简单的内容拼凑
- [ ] 有原创性内容
- [ ] 体现个人经验或判断

---

## 六、风格专属检查

### B 技术博客额外检查
- [ ] 代码/命令是否可运行？语法正确？
- [ ] 技术概念解释是否准确？
- [ ] 有没有实操体验或个人观点？
- [ ] 读起来像人写的还是AI生成的？
- [ ] 是否有至少一处踩坑/承认局限？

### D 评测对比额外检查
- [ ] 评测维度是否明确说明？
- [ ] 优缺点都说到了？
- [ ] 有明确结论（不是"各有优劣"收尾）？
- [ ] 有场景化推荐？

### E 观点洞察额外检查
- [ ] 有明确立场（不骑墙）？
- [ ] 用了真实事件/案例作锚点？
- [ ] 落到个人层面——"对你意味着什么"？
- [ ] 结尾不是空洞鸡汤？

---

## 七、互动性检查

- [ ] 文末有互动话题或问题
- [ ] 引导方式自然不生硬
- [ ] 互动话题与内容相关

---

## 八、技术规范检查

### 8.1 格式规范
- [ ] 手机端显示正常
- [ ] 图片加载正常
- [ ] 排版不错位
- [ ] 无乱码问题

### 8.2 封面图
- [ ] 封面图清晰美观
- [ ] 尺寸符合规范（21:9 封面、16:9 内容图）
- [ ] 与内容主题匹配
- [ ] 无水印

---

## 质量评分参考

| 等级 | 分数 | 标准 |
|------|------|------|
| 优秀 | 90-100 | 核心项全过，无瑕疵，有传播潜力 |
| 良好 | 80-89 | 核心项通过，1-2处小瑕疵，可发布 |
| 需改进 | 70-79 | 有明显问题，建议优化后发 |
| 不合格 | <70 | 存在严重问题，必须重写 |

- [ ] 无明显错别字
- [ ] 标题不是标题党
- [ ] 内容真实准确
- [ ] 逻辑基本通顺
- [ ] 手机端显示正常
